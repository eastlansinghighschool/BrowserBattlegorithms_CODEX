What Is Battlegorithms?

Browser Battlegorithms is a strategy game where teams race to grab the other side’s flag and bring it home. You can control human runners with the keyboard and teach allies what to do by connecting code blocks into a short program.

Board And Teams

Runner
Your moving game piece. Some runners are humans, some follow your program, and some are computer opponents.
Home side
The half of the field where your team starts and where your own flag belongs.
Enemy side
The half of the field where the other team starts and where their flag begins.
Frozen
A frozen runner cannot move for a short time.
Barrier
A temporary obstacle that blocks a lane until removed or expired.
Target square
A highlighted practice square used in some guided levels.

Guided Levels Vs Free Play

  • Guided Levels teach one idea at a time with a small challenge.
  • Free Play lets you choose teams, map, and opponent style.
  • Player vs Player gives both sides a human runner and a separate program tab.
  • Player vs CPU lets you test your ideas against easy or tactical computer teams.

Winning, Scoring, Flags, Freezing, Barriers, Jumping

  • Pick up the enemy flag by reaching its square.
  • Score by carrying that flag back to your home side.
  • You cannot stand on your own team’s flag while it is sitting at home.
  • Jump moves forward two spaces and is usually a one-time ability each round.
  • Place Barrier creates a forward obstacle if the space is legal.
  • Freeze Opponents affects nearby enemies for a short time.

How Block Programs Work

Your program runs once each time that runner gets a turn. Start with the event block, connect other blocks under it, and remember that the runner uses the first action it reaches in the chain.

If a condition block is true, the runner uses the action inside that branch. If the branch does not contain an action, the program keeps looking below.

Event Block

On Each Turn

This is the starting point for every runner program. Put your movement or condition blocks underneath it.

Movement Blocks

Move Forward

Move one space in your runner’s forward direction.

Move Backward

Move one space backward.

Move Up (screen)

Move one space upward on the screen.

Move Down (screen)

Move one space downward on the screen.

Move Randomly

Choose one random direction. Great for experiments, not for careful plans.

Move Toward

Step toward a chosen target such as the enemy flag, your base, the human runner, or the closest enemy.

Stay Still

Do not move. In some situations this can also clear a barrier directly in front.

Jump Forward

Leap forward over one space and land two spaces ahead if the landing space is open.

Place Barrier

Place a barrier in front of the runner if that space is legal.

Freeze Opponents

Freeze nearby opponents when the team freeze is ready.

Condition Blocks

If sensor matches

Check an object and a relationship, then run the action inside if it is true.

If sensor matches else

Choose one action when the check is true and a different action when it is false.

Quick shortcut for checking whether a barrier is directly ahead.

Barrier shortcut with an else branch.

If enemy in front

Shortcut for checking an enemy directly ahead.

If have enemy flag

Check whether this runner is already carrying the enemy flag.

If can jump

Check whether jump is still available.

Beginner Strategy

Make one clear plan.
Early levels usually work best when the runner keeps doing one simple thing every turn.
Watch the goal.
Sometimes you only need to reach a target square. Other times you need to carry a flag all the way home.
Remember the first action rule.
The runner uses the first action it reaches, so order matters.

Sensing And Reacting

  • Use board checks when the puzzle changes based on what is nearby.
  • Sensors are strongest when they lead to one clear reaction.
  • Move Toward is helpful on open boards, but it is not full pathfinding.

Using Special Abilities

  • Jump is best when one leap solves a real obstacle problem.
  • Barrier is strongest when it protects a lane or delays a carrier.
  • Freeze works best when another runner can take advantage of the pause right away.

Teamwork And Runner Roles

  • Use runner index to split one shared program into different jobs.
  • Attackers chase flags, defenders react to threats, and support runners escort carriers.
  • Shared programs become more powerful when each ally has a clear role.

Free Play Tips

  • Start with a team size of 2 or 3 before trying bigger teams.
  • Easy CPU is good for experiments. Tactical CPU is better for testing real strategy.
  • In two-player mode, use the team tabs to switch between each side’s program.
  • Try different maps when a strategy feels too simple or too cramped.
